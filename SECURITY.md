# Security policy

## Reporting a vulnerability

Please report suspected vulnerabilities **privately**, via GitHub's private
vulnerability reporting: open this repository's **Security** tab and click
**Report a vulnerability**. Do not open a public issue for anything you
believe is exploitable.

You can expect an acknowledgement within five business days. We'll work with
you on a fix in a private advisory and credit you when it's published,
unless you prefer otherwise.

## Supported versions

The mirror's `main` branch is the supported version — fixes land there via
the next sync from the internal repository.

## Scope

This policy covers the code in this repository (the `precis-mcp` open
package). The deployment hardening guidance operators should apply is in
[docs/deployment/security-model.md](docs/deployment/security-model.md).
